# 🚨 Fraud Detection Messenger

Fraud Detection Messenger — учебный проект защищённого мессенджера с модулем автоматического выявления мошеннических сообщений.

Основная идея проекта заключается в объединении системы обмена сообщениями и интеллектуального анализа текста. При отправке сообщения система выполняет его проверку и определяет вероятность наличия признаков мошенничества. В случае обнаружения подозрительной активности пользователю отображается предупреждение.

Проект разработан в рамках выпускной квалификационной работы по направлению «Фундаментальная информатика и информационные технологии».

## 📌 Основные возможности

* регистрация и авторизация пользователей;
* обмен сообщениями в режиме реального времени;
* поддержка WebSocket-соединений;
* анализ текстовых сообщений на наличие признаков мошенничества;
* отображение предупреждений о подозрительных сообщениях;
* отправка голосовых сообщений;
* автоматическая расшифровка голосовых сообщений;
* механизм пользовательских жалоб;
* подготовка данных для последующего дообучения модели.

## 🚀 Запуск проекта

### Через Docker

```bash
docker compose up --build
```

После запуска приложение будет доступно по адресам:

* Frontend: http://localhost
* Backend: http://localhost:8000

### Локальный запуск

Backend:

```bash
cd backend

pip install -r requirements.txt

python manage.py migrate
python manage.py runserver
```

Frontend:

```bash
cd frontend

npm install
npm run dev
```

## 🧠 Модуль обнаружения мошенничества

В проекте используется гибридный подход к анализу сообщений.

Rule-based модуль выполняет поиск формальных признаков мошенничества:

* подозрительные ссылки;
* признаки срочности;
* запросы денежных переводов;
* попытки получения конфиденциальных данных;
* типовые мошеннические шаблоны.

Нейросетевой модуль построен на основе модели DistilBERT и используется для анализа смыслового содержания сообщения.

Совместное использование двух подходов позволяет повысить качество классификации и уменьшить количество ложных срабатываний.

## 🎤 Голосовые сообщения

Для голосовых сообщений реализован следующий сценарий обработки:

1. Получение аудиозаписи.
2. Преобразование речи в текст.
3. Очистка полученной транскрипции.
4. Передача текста в модуль анализа.
5. Отображение результата пользователю.
